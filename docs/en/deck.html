Problem

  • One strong LLM alone = expensive, often overkill on simple tasks.
  • A naive MoE = dilutes the thesis on analytical tasks (multi-expert fusion kills the reasoning).
  • The real challenge is not "build a MoE" — it's knowing when to activate it.

Solution — adaptive orchestrator

The Genie classifies every query before routing:

  • Analytical mono-domain → delegation to 1 strong expert, no fusion (k=1).
  • Constructive multi-domain → top-k experts + editor-in-chief fusion (one narrator, not a patchwork).
  • Mode veto — confronts fusion with the best single expert and rejects its own fusion if single wins. (implemented, not yet benchmarked)

The claim is not "we have 40 experts" — the benchmarked roster is 5, top-k 3. It's "we know when to use only one."

Credible measurement methodology

  • Pairwise A/B randomized — anti position bias.
  • Double judge — qwen-max + qwen-plus, victory only if both agree (tie otherwise — judge noise acknowledged).
  • 4-criteria rubric (exactitude, complementation, depth, actionability) × 0–5 = /20.
  • 5 reps per question (up to 10), mean ± std. Baseline = qwen-turbo single agent, same prompt, same judge.

Product identity — Le Camp

  • L'Embûche — an adversarial strategist audits the gang's robustness and recruits a breakthrough expert.
  • Le Conciliabule — an MoE recruiter forms the optimal squad from a natural-language mission, then runs a recruitment debate.
  • Les Sceaux — generative SVG sigil + dynamic tide; every Genie has its own crest.
